## Phase 5: 修复 PE Array GlobalBuffer 读路径 (断开的数据通路)
### 问题诊断
- core_top.v 中 PE 的 gbuf_rd_en/gbuf_rd_addr 悬空未连接
- pe_top.v 中 A Buffer 和 B Buffer 写端口硬编码为 0（永远不会填充数据）
- pe_top.v 的 gbuf_rd_* 声明为 output reg 但从未驱动
- GlobalBuffer 只有一个读端口，被 task_loader 独占，缺少仲裁

### 修复内容 (3个文件, iverilog 编译通过)
1. **core_top.v**: 
   - 新增 STATE_LOAD_PE (在 LOAD_TASK 和 COMPUTE 之间)
   - GlobalBuffer 读端口 MUX (task_loader vs PE Array)
   - PE 顺序加载计数器 + pe_load_done/pe_load_all_done 信号
   - 为 PE 传入 M 维度 + SRAM 基地址 (a/b row/col/val sram)

2. **pe_top.v**: 
   - A/B buffer 写端口改为内部信号驱动 (非硬编码0)
   - 新增 10 状态 load FSM: LD_A_ROW → LD_A_DATA_COL → LD_A_DATA_VAL → LD_A_DATA_WR_V → LD_B_ROW → LD_B_DATA_C0/V0/C1/V1 → LD_DONE
   - A buffer 布局: [0..M] = A_row_ptr, [M+1..] = 交错 col/val
   - B buffer 布局: [0..K] = B_row_ptr (bank0), [K+1..] = 64-bit words {col1,val1,col0,val0}

3. **pe_decompress.v**:
   - S0_A_ELEM 拆分为 S0_A_ELEM_C (读col) + S0_A_ELEM_V (读val)
   - 新增 a_ptr_start 端口用于 A buffer 数据重映射
   - Lane dispatch 改为配对模式: 只用 lanes 0和2, (bank0,bank1)=(col,val)
   - 有效 N_MAC = 2 (原来的 N_MAC=4 减半, 每个 (col,val) 对占用2个bank)
   - S1 B buffer 地址: (K+1) + element_idx/2

## Phase 6: sp_elementwise 完整实现 + gbuf MUX 三路化
- 删除 `rtl/core/scheduler.v` (死代码，Phase 2 已旁路)
- core_top.v: gbuf MUX 扩展为 3 路 (task_loader | PE | elementwise)
- core_top.v: ADD/SUB 时跳过 STATE_LOAD_PE
- sp_elementwise.v: 完整重写 (row-by-row merge-sort, 读 col+val, ALU add/sub)
  - 接受 6 路 SRAM 基地址 (a_row/col/val, b_row/col/val)
  - 逐行处理: ROW_SETUP → MERGE_COL_A → MERGE_VAL_A → MERGE_COL_B → MERGE_VAL_B → COMPARE
  - merge-sort: 比较 col 索引，匹配时 add/sub，否则直接通过
  - SUB 时 B-only 元素取负 (0-B=-B)
  - 输出单元素流到 C CSR Writer (通过统一 MUX)
- iverilog 编译通过

## Phase 8: 全面 Bug 修复 (9 个功能正确性缺陷)
### core_top.v
- LOAD_TASK 拆分为 LOAD_TASK_DATA + PARSE_TASK (消除 load/task_loader 竞态)
- COMPUTE 期间并行启动 c_csr_writer (之前 WRITE_CSR 在 COMPUTE 后, PE 输出已消失)
- 新增 STATE_CSR_FINALIZE 替代 WRITE_CSR, 等 csr_done
### c_csr_writer.v
- 新增 compute_done 输入, start=COMPUTE 期间接收, compute_done+空转后 prefix sum
- prefix sum 补全 row_ptr[0..M] (之前缺最后一个 total nnz)
- 移除 row_ptr_base_addr/col_val_base_addr 端口
### pe_top.v
- 删除 ld_state_next always @(*) 块 (时序+组合双驱动)
- B data 写地址: ld_B_Kp1 + (ld_cnt >> 1) (之前直接 +ld_cnt, 浪费地址)
- 统一 A data base = M+1
### pe_decompress.v
- entry FIFO 加 row_id (52-bit), s1_cur_row 从 entry 读取而非固定 row_start
- done = pipe_empty && s0_all_rows_done && agg_idle (等 aggregation drain)
- b_buf_rd_en/addr 双驱动修复: S0/S1 合并为一个 always 块
### pe_aggregation.v
- 多 lane 竞态修复: 串行化处理, 每个周期只处理一个有效 lane
- 新增 agg_idle 输出 (无 active row + 无 outputting + 无 in-flight)
- 移除 conflict_detection (串行化后不需要)

## Phase 10: 外层数据流全面修复 (7 个严重 bug)
### defines.vh
- 新增 `DATA_BYTE_LOG2=1`, `AXI_ELEM_PER_BEAT_LOG=5`

### load.v: beat 数计算修复
- `n_elem_per_beat_log = DATA_BITS_LOG2 + N_MAC_BITS` (4+3=7, 错误) → `AXI_ELEM_PER_BEAT_LOG` (5, 正确=log2(32))

### store.v: 地址单位修复
- `total_bytes = xsize << DATA_BITS_LOG2` (×16) → `xsize << DATA_BYTE_LOG2` (×2)
- `osp_rd_addr` 固定 saddr → `saddr + (wcnt << 5)` 逐 beat 递进 32 元素
- `wcnt` 在 STATE_IDLE 时复位

### pe_aggregation.v: reset 分支修复
- 添加缺失的 `end else begin` (正常逻辑之前在 reset 黑分支里, 永不执行)

### task_loader.v + core_top.v: task base 修复
- task_loader 新增 `task_sram_base` 输入, 替代硬编码 `base_offset <= 0`
- core_top 从 LOAD_TASK 指令 latch sram_offset 并传入

### fetch.v: 取指完成条件修复
- `queue_count >= ilen` → `rvalid && rready && rlast` (解决单 beat 时早跳)

### core_top.v: LOAD 状态机重构
- 删除 `load_sub_state`, 改为每指令独立状态 (`LOAD_A_ROW→COL→VAL→LOAD_B_ROW→COL→VAL`)
- 消除跨状态消耗多条 LOAD 指令的风险

### c_csr_writer + testbench: 32-bit row_ptr
- row_ptr 输出从 16-bit → 32-bit (2×16-bit 写入, 低字→高字)
- col/val 起始地址: M+1 → 2*(M+1)
- test_dut.py verify_results 适配 (32-bit 解析)
