同志，这里我建议你先按**“能写出来、能仿真、能逐步扩展”**的原则设计 PE。不要一上来就追求满吞吐，否则你会同时遇到 B 读取带宽、4-MAC 发射、product FIFO 多写、accumulator 多端口冲突、FP16 流水相关冒险等问题。

赛题说明里要求矩阵元素采用标准 FL16，A/B/C 存在 FPGA SRAM 中且 C 与 A/B 在不同 bank，这和我们现在的 `PE 本地 A/B buffer + 全局 C_dense_buffer` 方向是兼容的。

---

# 1. PE 内部推荐实现版本

我建议 PE 第一版采用：

```text
A row iterator
    ↓
B row streamer
    ↓
task_packer
    ↓
task_group_fifo
    ↓
4 × FP16 multiplier
    ↓
product_group_fifo
    ↓
product_serializer
    ↓
serial accumulator
    ↓
acc_buffer[512]
    ↓
row_writeback
```

也就是：

```text
不规则 A/B 稀疏访问
    ↓
变成 task stream
    ↓
打包成 4-lane task group
    ↓
喂给 4 个 MAC
    ↓
结果串行累加到 acc_buffer
```

第一版 accumulator 仍然串行累加。这样 4 个 MAC 的输出会被 product FIFO 缓冲，后端慢一点没关系，先保证正确。

---

# 2. PE 内部需要哪些 buffer / reg

假设参数：

```verilog
localparam MAX_DIM            = 512;
localparam A_ROW_SLOT_PER_PE  = 128;
localparam A_NNZ_SLOT_PER_PE  = 16384;
localparam B_ROW_SLOT         = 512;
localparam B_NNZ_SLOT         = 78848;
localparam N_MAC              = 4;
localparam DATA_WIDTH         = 16;   // FP16
localparam IDX_WIDTH          = 16;
localparam OFFSET_WIDTH       = 32;
```

---

## 2.1 A buffer

每个 PE 保存自己的 A_group：

```verilog
reg [63:0] A_row_desc_buf [0:A_ROW_SLOT_PER_PE-1];
reg [15:0] A_col_buf      [0:A_NNZ_SLOT_PER_PE-1];
reg [15:0] A_val_buf      [0:A_NNZ_SLOT_PER_PE-1];
```

每个 `A_row_desc` 建议 64 bit：

```text
A_row_desc[15:0]   = global_row_id
A_row_desc[31:16]  = row_nnz
A_row_desc[63:32]  = start_offset
```

也就是：

```verilog
wire [15:0] desc_global_row = row_desc_reg[15:0];
wire [15:0] desc_row_nnz    = row_desc_reg[31:16];
wire [31:0] desc_start      = row_desc_reg[63:32];
```

容量：

```text
A_row_desc = 128 × 64 bit = 1 KB
A_col      = 16384 × 16 bit = 32 KB
A_val      = 16384 × 16 bit = 32 KB
A_group 总计约 65 KB / PE
```

---

## 2.2 B buffer

每个 PE 保存一份完整 B 的按行 compact 格式：

```verilog
reg [63:0] B_row_desc_buf [0:B_ROW_SLOT-1];
reg [15:0] B_col_buf      [0:B_NNZ_SLOT-1];
reg [15:0] B_val_buf      [0:B_NNZ_SLOT-1];
```

每个 `B_row_desc` 建议：

```text
B_row_desc[15:0]   = row_nnz
B_row_desc[31:16]  = reserved
B_row_desc[63:32]  = start_offset
```

读取：

```verilog
wire [15:0] b_desc_nnz   = b_row_desc_reg[15:0];
wire [31:0] b_desc_start = b_row_desc_reg[63:32];
```

容量：

```text
B_row_desc = 512 × 64 bit ≈ 4 KB
B_col      = 78848 × 16 bit ≈ 154 KB
B_val      = 78848 × 16 bit ≈ 154 KB
B 总计约 312 KB / PE
```

这个是 PE 最大的片上资源开销。

---

## 2.3 task packer / task FIFO

单个乘法 task 的信息是：

```text
col_id
a_val
b_val
```

建议 64-bit 对齐：

```text
task[15:0]   = col_id
task[31:16]  = a_val
task[47:32]  = b_val
task[63:48]  = reserved
```

因为你有 4 个 MAC，所以不要让 FIFO 每次只弹一个 task。更推荐设计成 **task_group_fifo**：

```text
一个 task_group = 4 个 task + 4 bit valid
```

可以定义成 256 bit：

```text
task_group[3:0]      = lane_valid
task_group[67:4]     = lane0 task
task_group[131:68]   = lane1 task
task_group[195:132]  = lane2 task
task_group[259:196]  = lane3 task
```

为了 Verilog 简单，可以直接用 260 bit：

```verilog
localparam TASK_WIDTH       = 64;
localparam TASK_GROUP_WIDTH = 4 + 4*TASK_WIDTH; // 260
```

FIFO 容量建议：

```text
task_group_fifo depth = 16 或 32
```

容量：

```text
16 × 260 bit ≈ 520 B
32 × 260 bit ≈ 1 KB
```

不大，可以用 LUTRAM 或 BRAM。

---

## 2.4 product FIFO

MAC 输出需要带列号：

```text
product[15:0]  = col_id
product[31:16] = product_val
```

如果 product 仍然用 FP16：

```verilog
localparam PRODUCT_WIDTH = 32;
```

4 个 MAC 一次输出一个 product group：

```verilog
localparam PRODUCT_GROUP_WIDTH = 4 + 4*PRODUCT_WIDTH; // 132
```

product_group_fifo depth 建议：

```text
32 或 64
```

容量：

```text
32 × 132 bit ≈ 528 B
64 × 132 bit ≈ 1 KB
```

---

## 2.5 acc_buffer

当前 PE 一次只处理一个 C 行，所以需要：

```verilog
reg [15:0] acc_buf [0:511];
```

容量：

```text
512 × 16 bit = 1 KB
```

如果以后用 FP32 累加：

```verilog
reg [31:0] acc_buf [0:511];
```

容量：

```text
512 × 32 bit = 2 KB
```

第一版为了简单，先用 FP16 累加也可以。

# 3. PE 的主要寄存器

下面这些 reg 是 PE 控制流必须有的。

---

## 3.1 行级控制寄存器

```verilog
reg [7:0]  row_idx;          // 当前 PE 本地第几行，最多 128 行
reg [15:0] row_count;        // 当前 PE 实际处理多少行

reg [63:0] row_desc_reg;
reg [15:0] cur_global_row;
reg [15:0] cur_a_row_nnz;
reg [31:0] cur_a_start;
```

---

## 3.2 A iterator 寄存器

```verilog
reg [15:0] a_nnz_left;
reg [31:0] a_ptr;

reg [15:0] cur_k;
reg [15:0] cur_a_val;
```

含义：

```text
a_ptr       当前 A_col/A_val 读到 compact 数组的哪个位置
a_nnz_left  当前 A 行还剩多少个 A 非零没处理
cur_k       当前 A(i,k) 的 k
cur_a_val   当前 A(i,k) 的值
```

---

## 3.3 B streamer 寄存器

```verilog
reg [63:0] b_row_desc_reg;
reg [31:0] b_ptr;
reg [15:0] b_nnz_left;

reg [15:0] cur_b_col;
reg [15:0] cur_b_val;
```

含义：

```text
b_ptr       当前 B_col/B_val 读到 compact 数组的哪个位置
b_nnz_left  当前 B(k,:) 还剩多少个非零没生成 task
```

---

## 3.4 task packer 寄存器

```verilog
reg [3:0]  pack_valid;
reg [63:0] pack_task0;
reg [63:0] pack_task1;
reg [63:0] pack_task2;
reg [63:0] pack_task3;
reg [1:0]  pack_count;
```

含义：

```text
pack_task0~3 是正在凑的一组 4-lane task
pack_count 表示当前已经凑了几个 task
pack_valid 表示哪些 lane 有效
```

---

## 3.5 product serializer 寄存器

```verilog
reg [3:0]  prod_valid_buf;
reg [31:0] prod_buf0;
reg [31:0] prod_buf1;
reg [31:0] prod_buf2;
reg [31:0] prod_buf3;
reg [1:0]  prod_lane_ptr;
```

如果 product_group_fifo 输出一个 4-lane product group，serializer 每次取一个 lane 送给 accumulator。

---

## 3.6 accumulator 寄存器

如果你第一版每次只处理一个 product，且等待 FP16 adder 完成再处理下一个 product，可以不需要复杂 scoreboard。

需要：

```verilog
reg [15:0] acc_col_reg;
reg [15:0] acc_old_reg;
reg [15:0] acc_delta_reg;
reg [15:0] acc_new_reg;

reg        acc_busy;
reg [3:0]  acc_wait_cnt;
```

如果 FP16 adder 是组合逻辑，这更简单。
如果 FP16 adder 是多拍流水，就要根据 latency 等待。

# 4. PE 顶层 FSM

建议 PE 状态：

```verilog
localparam PE_IDLE              = 4'd0;
localparam PE_LOAD_ROW_DESC     = 4'd1;
localparam PE_CLEAR_ACC         = 4'd2;
localparam PE_LOAD_A_ELEM       = 4'd3;
localparam PE_LOAD_B_DESC       = 4'd4;
localparam PE_STREAM_B_ROW      = 4'd5;
localparam PE_FLUSH_TASK_PACK   = 4'd6;
localparam PE_WAIT_TASK_DRAIN   = 4'd7;
localparam PE_WAIT_PRODUCT_DRAIN= 4'd8;
localparam PE_WRITE_ROW         = 4'd9;
localparam PE_NEXT_ROW          = 4'd10;
localparam PE_DONE              = 4'd11;
```

主流程：

```text
PE_IDLE
  等 start

PE_LOAD_ROW_DESC
  读取 A_row_desc[row_idx]

PE_CLEAR_ACC
  清空 acc_buf[0:511]

PE_LOAD_A_ELEM
  读取当前 A(i,k)

PE_LOAD_B_DESC
  用 k 读取 B_row_desc[k]

PE_STREAM_B_ROW
  读取 B(k,:) 的每个 B(k,j)，生成 task

PE_FLUSH_TASK_PACK
  当前 A 行 task 生成完，如果 packer 里有不足 4 个 task，flush 成 partial group

PE_WAIT_TASK_DRAIN
  等 task_group_fifo 空，MAC 不再有任务

PE_WAIT_PRODUCT_DRAIN
  等 product_group_fifo / serializer / accumulator 全部空

PE_WRITE_ROW
  写 acc_buf 到 C_dense_buffer[global_row_id][:]

PE_NEXT_ROW
  row_idx++，处理下一行

PE_DONE
  所有行完成
```

---

# 5. 任务生成：A 如何获取 B 任务

这段是 PE 的核心。

逻辑上是：

```text
当前 A 行：
for 每个 A(i,k):
    读取 B_row_desc[k]
    for 每个 B(k,j):
        task = {j, A(i,k), B(k,j)}
        送入 task_packer
```

Verilog 风格可以这样写。

---

## 5.1 读取 A 行描述符

```verilog
always @(posedge clk) begin
    if (rst) begin
        row_desc_reg <= 64'd0;
    end else if (state == PE_LOAD_ROW_DESC) begin
        row_desc_reg    <= A_row_desc_buf[row_idx];

        cur_global_row  <= A_row_desc_buf[row_idx][15:0];
        cur_a_row_nnz   <= A_row_desc_buf[row_idx][31:16];
        cur_a_start     <= A_row_desc_buf[row_idx][63:32];

        a_ptr           <= A_row_desc_buf[row_idx][63:32];
        a_nnz_left      <= A_row_desc_buf[row_idx][31:16];
    end
end
```

---

## 5.2 读取当前 A(i,k)

```verilog
always @(posedge clk) begin
    if (rst) begin
        cur_k     <= 16'd0;
        cur_a_val <= 16'd0;
    end else if (state == PE_LOAD_A_ELEM) begin
        cur_k     <= A_col_buf[a_ptr];
        cur_a_val <= A_val_buf[a_ptr];
    end
end
```

读取后去 `PE_LOAD_B_DESC`。

---

## 5.3 用 k 读取 B_row_desc[k]

```verilog
always @(posedge clk) begin
    if (rst) begin
        b_row_desc_reg <= 64'd0;
        b_ptr          <= 32'd0;
        b_nnz_left     <= 16'd0;
    end else if (state == PE_LOAD_B_DESC) begin
        b_row_desc_reg <= B_row_desc_buf[cur_k];

        b_ptr          <= B_row_desc_buf[cur_k][63:32];
        b_nnz_left     <= B_row_desc_buf[cur_k][15:0];
    end
end
```

---

## 5.4 读取 B(k,j)，生成 task

```verilog
wire task_packer_ready;

reg        task_in_valid;
reg [63:0] task_in_data;

always @(*) begin
    task_in_valid = 1'b0;
    task_in_data  = 64'd0;

    if (state == PE_STREAM_B_ROW && b_nnz_left != 0 && task_packer_ready) begin
        task_in_valid = 1'b1;

        task_in_data[15:0]   = B_col_buf[b_ptr];  // col_id = j
        task_in_data[31:16]  = cur_a_val;         // a_val
        task_in_data[47:32]  = B_val_buf[b_ptr];  // b_val
        task_in_data[63:48]  = 16'd0;
    end
end
```

写入成功后更新：

```verilog
always @(posedge clk) begin
    if (rst) begin
        b_ptr      <= 32'd0;
        b_nnz_left <= 16'd0;
    end else if (state == PE_STREAM_B_ROW &&
                 b_nnz_left != 0 &&
                 task_packer_ready) begin
        b_ptr      <= b_ptr + 1;
        b_nnz_left <= b_nnz_left - 1;
    end
end
```

当 `b_nnz_left == 0`，说明当前 A(i,k) 对应的 B(k,:) 已经处理完了。然后：

```verilog
if (a_nnz_left > 1) begin
    a_ptr      <= a_ptr + 1;
    a_nnz_left <= a_nnz_left - 1;
    state      <= PE_LOAD_A_ELEM;
end else begin
    state      <= PE_FLUSH_TASK_PACK;
end
```

注意实际写 FSM 时，`a_ptr/a_nnz_left` 的更新最好放在状态跳转处统一写，避免多 always 驱动。

# 6. task_packer 怎么写

`task_packer` 接收单个 task，然后每凑满 4 个，写入 `task_group_fifo`。

接口：

```verilog
input  wire        task_in_valid;
output wire        task_in_ready;
input  wire [63:0] task_in_data;

output reg         group_wr_en;
output reg [259:0] group_wr_data;
input  wire        group_fifo_full;

input  wire        flush_pack;
```

`task_in_ready`：

```verilog
assign task_in_ready = !group_fifo_full;
```

核心逻辑：

```verilog
always @(posedge clk) begin
    if (rst) begin
        pack_count <= 2'd0;
        pack_valid <= 4'b0000;
        pack_task0 <= 64'd0;
        pack_task1 <= 64'd0;
        pack_task2 <= 64'd0;
        pack_task3 <= 64'd0;
        group_wr_en <= 1'b0;
        group_wr_data <= 260'd0;
    end else begin
        group_wr_en <= 1'b0;

        if (task_in_valid && task_in_ready) begin
            case (pack_count)
                2'd0: begin
                    pack_task0 <= task_in_data;
                    pack_valid[0] <= 1'b1;
                    pack_count <= 2'd1;
                end

                2'd1: begin
                    pack_task1 <= task_in_data;
                    pack_valid[1] <= 1'b1;
                    pack_count <= 2'd2;
                end

                2'd2: begin
                    pack_task2 <= task_in_data;
                    pack_valid[2] <= 1'b1;
                    pack_count <= 2'd3;
                end

                2'd3: begin
                    pack_task3 <= task_in_data;

                    if (!group_fifo_full) begin
                        group_wr_en <= 1'b1;

                        group_wr_data[3:0]       <= 4'b1111;
                        group_wr_data[67:4]      <= pack_task0;
                        group_wr_data[131:68]    <= pack_task1;
                        group_wr_data[195:132]   <= pack_task2;
                        group_wr_data[259:196]   <= task_in_data;

                        pack_count <= 2'd0;
                        pack_valid <= 4'b0000;
                    end
                end
            endcase
        end

        // 当前 A 行所有 task 都生成完，但 packer 里还有 1~3 个 task
        if (flush_pack && pack_count != 0 && !group_fifo_full) begin
            group_wr_en <= 1'b1;

            group_wr_data[3:0]       <= pack_valid;
            group_wr_data[67:4]      <= pack_task0;
            group_wr_data[131:68]    <= pack_task1;
            group_wr_data[195:132]   <= pack_task2;
            group_wr_data[259:196]   <= pack_task3;

            pack_count <= 2'd0;
            pack_valid <= 4'b0000;
        end
    end
end
```

这个 packer 的意义就是：

```text
B(k0,:) 剩 2 个 task
B(k1,:) 又来了 2 个 task
四个 task 会被打包进同一个 group
```

从而填满 4 个 MAC。

# 7. task_group_fifo 如何分发给 4 个 MAC

当 `task_group_fifo` 非空，且 product FIFO 有空间，就读取一个 group：

```verilog
wire [259:0] group_rd_data;
wire         group_fifo_empty;
reg          group_rd_en;

wire [3:0]  mac_valid;
wire [63:0] mac_task0, mac_task1, mac_task2, mac_task3;

assign mac_valid = group_rd_data[3:0];

assign mac_task0 = group_rd_data[67:4];
assign mac_task1 = group_rd_data[131:68];
assign mac_task2 = group_rd_data[195:132];
assign mac_task3 = group_rd_data[259:196];
```

MAC 输入拆分：

```verilog
wire [15:0] mac0_col = mac_task0[15:0];
wire [15:0] mac0_a   = mac_task0[31:16];
wire [15:0] mac0_b   = mac_task0[47:32];

wire [15:0] mac1_col = mac_task1[15:0];
wire [15:0] mac1_a   = mac_task1[31:16];
wire [15:0] mac1_b   = mac_task1[47:32];

// mac2 / mac3 同理
```

读取控制：

```verilog
always @(*) begin
    group_rd_en = 1'b0;

    if (!group_fifo_empty && !product_group_fifo_full) begin
        group_rd_en = 1'b1;
    end
end
```

# 8. 4 个 MAC 怎么写

你可以先把 FP16 乘法器封装成模块：

```verilog
module fp16_mul (
    input  wire        clk,
    input  wire        rst,
    input  wire        in_valid,
    input  wire [15:0] a,
    input  wire [15:0] b,
    output reg         out_valid,
    output reg  [15:0] y
);
    // 第一版可以先用仿真模型或者 IP core 替换
endmodule
```

PE 里例化 4 个：

```verilog
fp16_mul u_mul0 (
    .clk(clk),
    .rst(rst),
    .in_valid(mac_valid[0]),
    .a(mac0_a),
    .b(mac0_b),
    .out_valid(mul_valid[0]),
    .y(mul_y0)
);

fp16_mul u_mul1 (...);
fp16_mul u_mul2 (...);
fp16_mul u_mul3 (...);
```

注意：`col_id` 要跟着乘法 pipeline 延迟同步。

如果乘法器 latency 是 `MUL_LAT`，需要：

```verilog
reg [15:0] col_pipe0 [0:MUL_LAT-1];
reg        val_pipe0 [0:MUL_LAT-1];
```

示意：

```verilog
integer s;
always @(posedge clk) begin
    if (rst) begin
        for (s = 0; s < MUL_LAT; s = s + 1) begin
            col_pipe0[s] <= 16'd0;
            val_pipe0[s] <= 1'b0;
        end
    end else begin
        col_pipe0[0] <= mac0_col;
        val_pipe0[0] <= mac_valid[0];

        for (s = 1; s < MUL_LAT; s = s + 1) begin
            col_pipe0[s] <= col_pipe0[s-1];
            val_pipe0[s] <= val_pipe0[s-1];
        end
    end
end

wire [15:0] mul0_col_out = col_pipe0[MUL_LAT-1];
wire        mul0_valid_out = val_pipe0[MUL_LAT-1];
```

最后 product 是：

```verilog
product0 = {mul0_col_out, mul_y0};
```

# 9. product_group_fifo 怎么接 MAC 输出

MAC 输出也打成 group：

```verilog
reg [131:0] product_group_data;
reg         product_group_wr_en;

always @(*) begin
    product_group_wr_en = 1'b0;
    product_group_data  = 132'd0;

    if ((|mul_valid) && !product_group_fifo_full) begin
        product_group_wr_en = 1'b1;

        product_group_data[3:0]      = mul_valid;

        product_group_data[35:4]     = {mul0_col_out, mul_y0};
        product_group_data[67:36]    = {mul1_col_out, mul_y1};
        product_group_data[99:68]    = {mul2_col_out, mul_y2};
        product_group_data[131:100]  = {mul3_col_out, mul_y3};
    end
end
```

这里 product entry 是：

```text
{col_id[15:0], product_val[15:0]}
```

共 32 bit。

# 10. product_serializer 怎么写

product_group_fifo 每次出来一组最多 4 个 product。
accumulator 一次只处理一个 product。
所以需要 serializer。

```verilog
reg [131:0] prod_group_reg;
reg [3:0]   prod_valid_reg;
reg [1:0]   prod_lane_ptr;
reg         prod_group_loaded;

reg         acc_in_valid;
wire        acc_in_ready;
reg [31:0]  acc_in_data;
```

加载 product group：

```verilog
always @(posedge clk) begin
    if (rst) begin
        prod_group_loaded <= 1'b0;
        prod_lane_ptr <= 2'd0;
    end else begin
        if (!prod_group_loaded && !product_group_fifo_empty) begin
            prod_group_reg    <= product_group_fifo_rd_data;
            prod_valid_reg    <= product_group_fifo_rd_data[3:0];
            prod_lane_ptr     <= 2'd0;
            prod_group_loaded <= 1'b1;
            product_group_fifo_rd_en <= 1'b1;
        end else begin
            product_group_fifo_rd_en <= 1'b0;
        end
    end
end
```

输出一个 lane 给 accumulator：

```verilog
always @(*) begin
    acc_in_valid = 1'b0;
    acc_in_data  = 32'd0;

    if (prod_group_loaded) begin
        case (prod_lane_ptr)
            2'd0: begin
                acc_in_valid = prod_valid_reg[0];
                acc_in_data  = prod_group_reg[35:4];
            end
            2'd1: begin
                acc_in_valid = prod_valid_reg[1];
                acc_in_data  = prod_group_reg[67:36];
            end
            2'd2: begin
                acc_in_valid = prod_valid_reg[2];
                acc_in_data  = prod_group_reg[99:68];
            end
            2'd3: begin
                acc_in_valid = prod_valid_reg[3];
                acc_in_data  = prod_group_reg[131:100];
            end
        endcase
    end
end
```

推进 lane：

```verilog
always @(posedge clk) begin
    if (rst) begin
        prod_lane_ptr <= 2'd0;
        prod_group_loaded <= 1'b0;
    end else if (prod_group_loaded && acc_in_ready) begin
        if (prod_lane_ptr == 2'd3) begin
            prod_group_loaded <= 1'b0;
            prod_lane_ptr <= 2'd0;
        end else begin
            prod_lane_ptr <= prod_lane_ptr + 1'b1;
        end
    end
end
```

如果某个 lane invalid，也可以跳过；第一版简单点可以仍然走 4 个 lane，其中 invalid lane 不让 accumulator 接收。

# 11. accumulator 怎么写

第一版最稳的 accumulator：**一次只处理一个 product，处理完再取下一个**。

这样不需要 `acc_busy[512]` scoreboard，因为不会有多个同列更新同时在飞。

输入：

```verilog
wire        acc_in_valid;
output wire acc_in_ready;
wire [31:0] acc_in_data;
```

拆分：

```verilog
wire [15:0] acc_in_col = acc_in_data[31:16];
wire [15:0] acc_in_val = acc_in_data[15:0];
```

但注意你上面的 product group 如果 `{col, product}` 排列是 `{col_id, product_val}`，要统一位段。建议固定：

```text
product[15:0]  = product_val
product[31:16] = col_id
```

那么：

```verilog
wire [15:0] product_val = acc_in_data[15:0];
wire [15:0] product_col = acc_in_data[31:16];
```

accumulator FSM：

```verilog
localparam ACC_IDLE  = 2'd0;
localparam ACC_ADD   = 2'd1;
localparam ACC_WRITE = 2'd2;

reg [1:0] acc_state;
reg [15:0] acc_col_reg;
reg [15:0] acc_old_reg;
reg [15:0] acc_delta_reg;
reg [15:0] acc_new_reg;
```

如果 FP16 add 是组合模块：

```verilog
wire [15:0] fp16_add_y;

fp16_add u_add (
    .a(acc_old_reg),
    .b(acc_delta_reg),
    .y(fp16_add_y)
);
```

控制：

```verilog
assign acc_in_ready = (acc_state == ACC_IDLE);

always @(posedge clk) begin
    if (rst) begin
        acc_state <= ACC_IDLE;
        acc_col_reg <= 16'd0;
        acc_old_reg <= 16'd0;
        acc_delta_reg <= 16'd0;
        acc_new_reg <= 16'd0;
    end else begin
        case (acc_state)
            ACC_IDLE: begin
                if (acc_in_valid) begin
                    acc_col_reg   <= product_col;
                    acc_delta_reg <= product_val;
                    acc_old_reg   <= acc_buf[product_col[8:0]];
                    acc_state     <= ACC_ADD;
                end
            end

            ACC_ADD: begin
                acc_new_reg <= fp16_add_y;
                acc_state   <= ACC_WRITE;
            end

            ACC_WRITE: begin
                acc_buf[acc_col_reg[8:0]] <= acc_new_reg;
                acc_state <= ACC_IDLE;
            end
        endcase
    end
end
```

如果 FP16 add 是多拍 IP，就把 `ACC_ADD` 拆成 `ACC_WAIT_ADD`，等待 `add_out_valid`。

# 12. acc_buffer 清零怎么写

每处理一个新 C 行，都要清空：

```verilog
reg [8:0] clear_idx;

always @(posedge clk) begin
    if (rst) begin
        clear_idx <= 9'd0;
    end else if (state == PE_CLEAR_ACC) begin
        acc_buf[clear_idx] <= 16'd0;
        clear_idx <= clear_idx + 1'b1;
    end
end
```

状态跳转：

```verilog
if (state == PE_CLEAR_ACC && clear_idx == 9'd511) begin
    next_state = PE_LOAD_A_ELEM;
end
```

这个需要 512 个周期。第一版可以接受。

# 13. 当前行什么时候算完

当前行完成必须满足：

```text
A 当前行全部非零处理完
B 当前行 streamer 全部完成
task_packer 已 flush
task_group_fifo 空
MAC pipeline 空
product_group_fifo 空
serializer 空
accumulator idle
```

可以定义：

```verilog
wire task_done =
    row_task_gen_done &&
    task_group_fifo_empty &&
    mac_pipeline_empty &&
    product_group_fifo_empty &&
    !prod_group_loaded &&
    (acc_state == ACC_IDLE);
```

只有 `task_done` 为 1，才能进入：

```text
PE_WRITE_ROW
```

# 14. 写 C_dense_buffer 怎么写

每行写 512 个 FP16：

```verilog
reg [8:0] write_col;

assign cbuf_wr_en   = (state == PE_WRITE_ROW);
assign cbuf_wr_addr = (cur_global_row << 9) + write_col;
assign cbuf_wr_data = acc_buf[write_col];
```

状态内更新：

```verilog
always @(posedge clk) begin
    if (rst) begin
        write_col <= 9'd0;
    end else if (state == PE_WRITE_ROW) begin
        if (cbuf_wr_ready) begin
            write_col <= write_col + 1'b1;
        end
    end else begin
        write_col <= 9'd0;
    end
end
```

跳转：

```verilog
if (state == PE_WRITE_ROW && write_col == 9'd511 && cbuf_wr_ready) begin
    next_state = PE_NEXT_ROW;
end
```

如果多 PE 同时写 C_dense_buffer，要接 arbiter：

```verilog
cbuf_wr_valid
cbuf_wr_ready
cbuf_wr_addr
cbuf_wr_data
```

单 PE 版本可以先直接写。

# 15. 一个简化版 PE 顶层端口建议

```verilog
module pe_spgemm_mul #(
    parameter A_ROW_SLOT_PER_PE = 128,
    parameter A_NNZ_SLOT_PER_PE = 16384,
    parameter B_ROW_SLOT        = 512,
    parameter B_NNZ_SLOT        = 78848
)(
    input  wire        clk,
    input  wire        rst,

    input  wire        start,
    input  wire [15:0] row_count,
    output reg         done,

    // loader 写 A buffer
    input  wire        a_desc_we,
    input  wire [7:0]  a_desc_waddr,
    input  wire [63:0] a_desc_wdata,

    input  wire        a_col_we,
    input  wire [13:0] a_col_waddr,
    input  wire [15:0] a_col_wdata,

    input  wire        a_val_we,
    input  wire [13:0] a_val_waddr,
    input  wire [15:0] a_val_wdata,

    // loader 写 B buffer
    input  wire        b_desc_we,
    input  wire [8:0]  b_desc_waddr,
    input  wire [63:0] b_desc_wdata,

    input  wire        b_col_we,
    input  wire [16:0] b_col_waddr,
    input  wire [15:0] b_col_wdata,

    input  wire        b_val_we,
    input  wire [16:0] b_val_waddr,
    input  wire [15:0] b_val_wdata,

    // 写 C dense buffer
    output wire        cbuf_wr_valid,
    input  wire        cbuf_wr_ready,
    output wire [17:0] cbuf_wr_addr,  // 512*512 = 262144，需要18 bit
    output wire [15:0] cbuf_wr_data
);
```

地址位宽：

```text
A_NNZ_SLOT_PER_PE = 16384 → 14 bit
B_NNZ_SLOT = 78848 → 17 bit
C_dense = 512×512 = 262144 → 18 bit
```

# 16. 最终建议：先写哪个版本

你现在让 agent 写代码，我建议明确告诉它：

第一版不要直接写满性能版，先写：

```text
1. 单 PE；
2. 可以保留 4-MAC 结构，但 accumulator 串行；
3. task_packer 打包 4 task；
4. product_group_fifo 缓冲 MAC 输出；
5. product_serializer 串行送 accumulator；
6. acc_buffer 写 C_dense_buffer；
7. 先用 FP16 add/mul wrapper，仿真时可以先用简化模型。
```

最小闭环必须先验证：

```text
A_row_desc → A_col/A_val → B_row_desc → B_col/B_val
→ task_packer → 4 MAC
→ product_fifo → acc_buffer
→ C_dense_buffer
```

等这个闭环正确后，再优化：

```text
1. B 4-wide read；
2. product_fifo 4-wide push；
3. banked accumulator；
4. 多 PE 并行写 C_dense_buffer。
```

一句话总结：

> PE 的硬件实现不要把“不规则 B 行访问”和“4 个 MAC 发射”硬绑在一起，而是用 task_packer 和 task_group_fifo 把不规则稀疏乘法任务整理成 4-lane 任务组；MAC 输出再用 product_group_fifo 和 serializer 串行累加到 acc_buffer。这样 Verilog 容易写，数据流清晰，也容易用波形调试。
